## All findings (full detail)

### Security

**B-1 | HIGH | Pre-existing**  
`plugins/ca/tools/farm.ts:720`  
`git add -A` in farm dispatcher bypasses the H-03 explicit-staging guard. Farm runs outside Claude's hook pipeline so the guard never fires. Remediation: replace with `git add <worker.filesWritten>`.

**B-2 | HIGH | Pre-existing**  
`plugins/ca/tools/farm.ts:626-627`  
`t.id` from user-supplied `plan.json` used unsanitized in git branch name and `path.resolve(ENV.worktreeRoot, t.id)`. Crafted `..` or path separator can escape worktree root. Remediation: restrict `t.id` to `[A-Za-z0-9._-]` max 64 chars in `validate()`.

**D-2 | MEDIUM | Pre-existing**  
`plugins/ca/tools/farm.ts:845`  
`plan.json` parsed without schema validation. `gate.commands`, `test.path`, `filesInScope`, `meta.apiBaseUrl` all from untrusted file. Malicious plan could inject arbitrary gate commands or redirect API client.

**D-3 | MEDIUM | Pre-existing**  
`plugins/ca/tools/farm.ts:299-300`  
Non-2xx API response body (≤500 chars) injected into `priorFailure`, fed back into next worker prompt. Limited prompt injection via error path.

**D-4 | MEDIUM | Pre-existing**  
`plugins/ca/hooks/_hooklib.py:108-116`  
`read_input()` returns `{}` on all parse failures (fail-open). Contradicts stated fail-loud principle. Defense-in-depth gap.

**N-1 | LOW | INTRODUCED BY CURRENT BRANCH**  
`plugins/ca/hooks/_prunelib.py:1082-1083`  
`transcript_path` from payload used without directory containment check on read path. Symlink guard on writes exists; read path is unguarded. Remediation: verify `transcript_path` resolves under `~/.claude/` in `hook_run()`.

**N-2 | LOW | Pre-existing**  
`plugins/ca/tools/farm.ts:131`  
`NOSIGN` constant disables GPG/SSH signing on all dispatcher commits. Unsigned commits not distinguished from legitimate commits in farm report. Remediation: documentation only.

**N-3 | LOW | Pre-existing**  
`plugins/ca/hooks/pre-bash.py:56`  
`LOG_TRUNC_RE` has known shell-spelling gaps (triple `>`, `exec 3>`). Add comment documenting known limitations.
