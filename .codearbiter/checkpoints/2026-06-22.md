## Blocks the 2.5.0 plugin release

### HIGH — `CRYPTO_RE` cannot see Node/TS TLS-disable
`plugins/ca/hooks/_hooklib.py:41-48`. `CRYPTO_RE` detects only Python `verify=False` /
`InsecureSkipVerify`; it misses `rejectUnauthorized: false` and `NODE_TLS_REJECT_UNAUTHORIZED`.
The H-09/H-09b crypto gate runs entirely off this regex, and all networked first-party code
(`farm.ts`, all of `ca-sandbox`) is TS. `security-controls.md` classifies `rejectUnauthorized:
false` as never-permitted, but the gate cannot enforce it. No live occurrence exists; this is an
enforcement-coverage gap in the shipped gate. **Fix before 2.5.0.**

## Fix-along (same family, bundled with the HIGH)

### MEDIUM — `SECRET_RE` only matches `=` assignment literals
`plugins/ca/hooks/_hooklib.py:49-52`. Misses colon/object forms (`"api_key": "..."`,
`apiKey: "..."`), env-file form, and AWS/`ghp_`/`sk-ant-` key prefixes — the forms that dominate
a TS/JSON repo. This is the `SECRET_RE`-assignment-literal item already deferred in
`open-questions.md`. The H-10/H-10b secret gate inherits the blind spot.

### MEDIUM — outbound prompt redactor has the same prefix gap
`plugins/ca/tools/farm.ts:264` (`SECRET_LINE`). Keyword-presence redactor with no AWS/`ghp_`/
high-entropy coverage. Mirror the `SECRET_RE` change so the hook gate and the outbound redactor
stay aligned.

### MEDIUM — `_hooklib.py` has no direct unit tests
`plugins/ca/hooks/_hooklib.py`. `CRYPTO_RE`/`SECRET_RE`/`is_*_path`/`marker_fresh`/digests are
exercised only indirectly. The HIGH fix needs a direct test anyway → create
`.github/scripts/test_hooklib.py`, which closes this finding too.
